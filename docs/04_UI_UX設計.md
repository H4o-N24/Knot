# UI/UX設計とDiscord機能の活用

## ユーザーフロー

### 1. 空き日の登録 (Monthly Setup)
- **トリガー**: 月末にBotが自動的に「来月の予定を登録してください」というメッセージを特定のチャンネルに投稿。
- **UI**: メッセージに配置された「予定を登録する」ボタンをクリック。
- **Action**: Discordの **Modal** が開き、カレンダー形式（または日付リスト）が表示される。ユーザーはタップで「空き」を選択。
- **API連携の布石**: Googleカレンダー連携済みユーザーには、Modal内に「Googleカレンダーから同期済み」というラベルを表示。

### 2. イベント作成 (Event Creation)
- **トリガー**: `/event create` コマンド。
- **UI**: コマンドの引数として「イベント名」「最低人数」「必須メンバー」を入力。
- **Action**: Botが計算を行い、**Select Menu** で「おすすめ候補日」を提示。
- **決定**: 主催者が候補から選択すると、イベントチャンネルが自動作成される。

### 3. 定員管理と自動繰り上げ (Participant Management)
- **UI**: イベントメッセージに「参加」「キャンセル」ボタンを配置。
- **ロジック**:
    - `max_participants` に達した後は「キャンセル待ち」として登録。
    - 既存参加者が「キャンセル」を押すと、`EventParticipants` テーブルの `joined_at` が最も古いキャンセル待ちユーザーを自動的に「参加」へ昇格させ、メンションで通知。

## デザインのこだわり
- **彩度の最適化**: Discordのダークモード/ライトモード両方で視認性が高く、かつ「少し彩度を抑えた」落ち着いた配色（ボタンのスタイルなど）を採用。
- **フィードバック**: 処理中には「思考中...」といったエフェメラルメッセージ（本人にしか見えないメッセージ）を出し、体感速度を向上。
